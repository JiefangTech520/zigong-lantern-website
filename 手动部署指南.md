# 自贡彩灯官网 - GitHub 部署指南

## 当前情况
- ✅ Git 仓库已初始化
- ✅ 文件已准备好
- ✅ GitHub 仓库已配置（远程地址）
- ❌ 网络环境限制：HTTPS 443 端口无法连接 GitHub

## 方案一：手动推送（推荐）

### 步骤 1：确认本地代码状态
```bash
cd "D:/AI/自贡彩灯官网"
git status
```

### 步骤 2：检查远程仓库
```bash
git remote -v
```
应该显示：
- 仓库地址：https://github.com/JiefangTech520/zigong-lantern-website.git
- 或：https://github.com/JiefangTech520/zigong-lanterns.git

### 步骤 3：临时关闭防火墙/代理尝试推送
```bash
# 方法 A：直接推送
git push -u origin main

# 方法 B：如果失败，尝试使用 SSH
# 首先生成 SSH 密钥（如果没有）
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 将 SSH 公钥添加到 GitHub 账户
# 路径：C:/Users/Administrator/.ssh/id_rsa.pub

# 然后切换远程地址为 SSH
git remote set-url origin git@github.com:JiefangTech520/zigong-lantern-website.git
git push -u origin main
```

### 步骤 4：配置 GitHub Pages
推送成功后：
1. 访问：https://github.com/JiefangTech520/zigong-lantern-website
2. 点击 Settings → Pages
3. Source 选择：Deploy from a branch
4. Branch 选择：main → / (root) → Save
5. 等待 1-2 分钟，网站即可访问：
   https://jiefangtech520.github.io/zigong-lantern-website/

## 方案二：使用其他网络环境

如果当前网络无法访问 GitHub：
1. 更换到其他网络（如手机热点）
2. 或者配置 HTTP/HTTPS 代理：
```bash
git config --global http.proxy http://proxy_address:port
git config --global https.proxy http://proxy_address:port
```

## 方案三：使用 GitHub Desktop

如果命令行推送失败，可以使用图形化工具：
1. 下载 GitHub Desktop：https://desktop.github.com/
2. 克隆仓库或添加本地仓库
3. 点击 "Publish repository" 推送到 GitHub

## 常见问题

### Q: 推送时提示 "authentication failed"
A: 检查 GitHub Token 是否有效，或使用 SSH 密钥认证

### Q: Pages 部署失败
A: 确保 index.html 在仓库根目录，检查文件名是否正确

### Q: 网站无法访问
A: 等待 2-3 分钟让 GitHub Pages 完成部署，然后刷新页面

## 技术支持

如需帮助，请检查：
1. GitHub Token 权限（需要 repo 权限）
2. 网络连接状态
3. 防火墙设置（允许 git.exe 或 git-remote-https.exe）
4. 代理设置（如企业网络需要配置代理）
